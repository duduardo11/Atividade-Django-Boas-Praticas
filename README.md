1. O que é versionamento de código?

    R. É o processo de controlar e registrar as alterações feitas no código ao longo do tempo, permitindo recuperar versões anteriores e acompanhar o histórico de mudanças.


2. Qual a função do Git no trabalho em equipe?

    R. O Git permite que várias pessoas trabalhem no mesmo projeto ao mesmo tempo sem sobrescrever o código umas das outras. Ele também mantém o histórico das alterações e facilita a organização do trabalho em equipe.


3. O que pode acontecer se a SECRET_KEY for exposta?

    R. Se a SECRET_KEY for exposta, a segurança da aplicação pode ser comprometida. Um invasor pode manipular sessões, falsificar autenticações e acessar dados sensíveis do sistema.


4. Qual o papel do arquivo .gitignore?

    R. O arquivo .gitignore serve para impedir que arquivos desnecessários ou sensíveis (como .env, venv e banco de dados) sejam enviados para o repositório no GitHub.
